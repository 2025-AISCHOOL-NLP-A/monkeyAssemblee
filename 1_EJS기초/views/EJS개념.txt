EJS란?
- EJS(Embeded Java Script) = HTML안에 JS를 섞어서 동작하게 만드는 "템플릿"엔진
- 템플릿엔진이란? 개발자는 페이지를 하나만 만들고(틀을 제공) 서버가 값을 동적으로 생성하는 엔진
    * Pug, Nunjucks, HandleBars... -> 고유한 문법이 존재 -> 문법을 새롭게 공부가 필요
    * EJS는 자바스크립트의 문법과 똑같다.
    1) HTML 기반으로 작성
    2) JS문법을 그대로 사용한다.
    3) 검색엔진에 최적화 (SEO)
- 사용처 : 로그인 후 메인페이지, 게시판 글 목록, 검색결과

[알아두면 좋은 웹 지식]
- EJS를 통해서 페이질 제작(동적생성) -> SSR(Server Side Rendering)
- SSR은 HTML파일을 서버가 제작 후 -> 클라이언트에게 보내는 방식
- SSR 장점
    1) 빠른 초기 로딩 속도 -> 클라이언트에서는 바로 출력이 가능
    2) SEO 최적화 : 검색엔진이 HTML파일을 읽어들 때 속도가 빠르다 -> 내용을 채워서 넘기기 때문에

- CSR(Client Side Rendering) ->  페이지를 제작하는 곳이 클라이언트
- 서버가 비어있는 html과 태그를 생성하는 JS코드를 클라이언트에게 전달 -> 클라이언트가 제작한다
    * Reat, Angular, Vue...
- CSR 장점
    1) 동적 UI 생성에 효율적이다. -> 특정 부분만 빠르고 쉽게 수정이 가능하다(비동기통신)
    2) 프론트엔드와 백엔드가 역할구분이 확실히 가능하다.

- 최신 트렌드 SSR + CSR 하이브리드 렌더링을 선호
- 초기페이지는 SSR기법 + 특정 내용을 바꿀 때는 CSR기법
* Next.js, Svelte
